# Solarflower Design Improvements — Solarpunk/Soil-Punk Vision

**Status:** Proposal
**Created:** March 2025
**Related:** `.claude/agents/00_design-guidelines.md`

---

These proposals push the Solarflower aesthetic toward a deeper solarpunk/soil-punk philosophy. They are aspirational enhancements that can be implemented incrementally. Each section includes reasoning and implementation guidance.

## 1. Expanded Color Palette: Earth & Dusk

### Rationale

The current palette is heavily sun-centric. A richer palette would deepen the solarpunk narrative — showing the cycle of day/night, the soil-to-sun connection, and the rhythms of nature. These colors extend the story beyond "sunlight" to encompass the full ecological cycle.

## 2. Organic Border Radii

### Rationale

Current radii are uniform geometric shapes. Soil-punk embraces imperfection — shapes that feel hand-formed, like potter's clay, or grown rather than manufactured. Organic radii create a sense of being "made by hand" rather than "generated by code."

### New Radius Variables

```css
:root {
  /* Existing preserved */
  --radius-sm: 6px;
  --radius-md: 12px;
  --radius-lg: 20px;
  --radius-xl: 28px;

  /* NEW: Organic shapes */

  /* Blob-like asymmetric radius */
  --radius-blob: 255px 15px 225px 15px / 15px 225px 15px 255px;

  /* Leaf shape — rounded on two sides */
  --radius-leaf: 20px 0px 20px 0px;

  /* Seed — organic oval */
  --radius-seed: 50% 50% 50% 50% / 60% 40% 60% 40%;

  /* Pebble — irregular circle */
  --radius-pebble: 65% 35% 55% 45% / 45% 55% 65% 35%;

  /* Pottery — hand-thrown feel */
  --radius-pot: 30px 70px 40px 60px / 60px 40px 70px 30px;
}
```

### Usage Guidelines

| Radius | Feel | Best Use |
|--------|------|----------|
| `--radius-blob` | Floating, organic blob | Decorative elements, blob shapes |
| `--radius-leaf` | Growing, directional | Cards pointing somewhere, arrows |
| `--radius-seed` | Beginning, contained | Small badges, seed/growth icons |
| `--radius-pot` | Crafted, hand-made | Buttons with "human" feel, containers |

### Example: Organic Card

```css
.card--organic {
  border-radius: var(--radius-pot);
  /* Or mix geometric with organic */
  border-radius: var(--radius-lg) var(--radius-blob);
}
```

## 4. Biophilic Micro-Interactions

### Rationale

Instead of generic animations (fade, slide, scale), create interactions that feel alive — growing, blooming, breathing. This connects users emotionally to the renewable energy mission by making the interface feel like a living system.

### Animation Concepts

#### Bloom Reveal

Cards "bloom" open on scroll reveal — starting folded and expanding:

```css
@keyframes bloom {
  0% {
    opacity: 0;
    transform: scale(0.8) translateY(20px);
    filter: blur(4px);
  }
  60% {
    transform: scale(1.02) translateY(-2px);
    filter: blur(0);
  }
  100% {
    opacity: 1;
    transform: scale(1) translateY(0);
  }
}
```

#### Breathing Button

Primary buttons gently pulse when idle, like a sleeping plant:

```css
@keyframes breathe {
  0%, 100% {
    box-shadow: 0 0 0 0 rgba(245, 166, 35, 0);
  }
  50% {
    box-shadow: 0 0 20px 4px rgba(245, 166, 35, 0.15);
  }
}

.btn--primary {
  animation: breathe 4s ease-in-out infinite;
}
.btn--primary:hover {
  animation: none;
  /* Immediate feedback on hover */
}
```

#### Growing Progress

Progress indicators as growing elements:

```css
@keyframes grow-up {
  from { transform: scaleY(0); transform-origin: bottom; }
  to { transform: scaleY(1); transform-origin: bottom; }
}

.progress-bar {
  transform-origin: bottom;
  animation: grow-up 1s var(--ease) forwards;
}
```

#### Root Connection

Loading states that look like roots spreading:

```css
@keyframes roots-spread {
  0% { transform: scaleX(0); opacity: 0; }
  50% { opacity: 1; }
  100% { transform: scaleX(1); opacity: 0.3; }
}
```

### Timing Guidelines

| Animation | Duration | Easing | Feel |
|-----------|----------|--------|------|
| Bloom reveal | 0.6–0.8s | var(--ease) | Opening flower |
| Breathe pulse | 4s | ease-in-out | Living, patient |
| Grow progress | 1s | var(--ease) | Natural growth |
| Root spread | 2s | ease-out | Spreading |

## 5. Organic Data Visualization

### Rationale

Solar data doesn't have to look like Excel spreadsheets. Data can be poetic — connecting users to natural rhythms rather than corporate dashboards. Transform abstract numbers into visual stories.

### Visualization Concepts

#### Sun Path as Flower Petal Diagram

Instead of a line chart, sun path could be rendered as a flower:

- Each hour is a petal
- Petal length = solar altitude
- Color shifts from dawn (pink) to noon (amber) to dusk (purple)
- The "center" is due south

```
         🌸
        /   \
   🌷  /     \  🌷
     \       /
      \     /
       \   /
        \ /
         +
    (south)
```

#### Energy Flow as Sap/Root System

Energy loss waterfall could be reimagined:

- Main "trunk" = gross potential
- Branches = each loss type
- Branch thickness = magnitude of loss
- Leaves at tips = final yield
- Color: green for productive, amber for losses

#### Monthly Yield as Tree Rings

Annual data could show as tree rings:

- Each ring = one month
- Ring thickness = yield that month
- Ring color = seasonal (green → amber → orange → bare)

#### Heatmaps with Organic Contours

Instead of rigid grid heatmaps:

- Use organic blob contours
- Smooth gradients between values
- Subtle animation of "shimmering" data
- Instead of squares use hexagons (associating for example bee hives)

### Implementation Example

```python
# Example: Sun path as polar "flower" plot
import plotly.graph_objects as go

fig = go.Figure()

# Petal for each season
for season, color, alpha in [('Spring', '#4CAF50', 0.6),
                             ('Summer', '#F5A623', 0.7),
                             ('Autumn', '#C75B39', 0.5)]:
    fig.add_trace(go.Scatterpolar(
        r=altitudes,
        theta=azimuths,
        fill='toself',
        fillcolor=color,
        line_color=color,
        opacity=alpha,
        name=season
    ))

fig.update_polars(
    radialaxis_visible=False,
    angularaxis_direction="clockwise",
    angularaxis_tickmode="array",
    angularaxis_ticktext=["N", "NE", "E", "SE", "S", "SW", "W", "NW"],
    angularaxis_tickvals=[0, 45, 90, 135, 180, 225, 270, 315]
)
```

## 6. Typography Evolution

### Rationale

Current system fonts are practical but forgettable. Solarpunk typography should feel like reading a botanical field guide or a hand-bound journal. Warm serifs add humanity; handwritten accents add character.

### Typography Additions

```css
:root {
  /* Existing preserved */
  --font-sans: "Inter", "Segoe UI", system-ui, ...;
  --font-mono: "SF Mono", "Fira Code", ...;

  /* NEW: Character fonts */

  /* Serif for headings — warm, editorial */
  --font-serif: "Lora", "Merriweather", "Georgia", serif;

  /* Handwritten for accents — personal, human */
  --font-hand: "Caveat", "Patrick Hand", "Indie Flower", cursive;
}
```

### Usage Guidelines

| Font | Feel | Use |
|------|------|-----|
| `--font-serif` | Editorial, authoritative, warm | H1, H2, article titles, feature headings |
| `--font-hand` | Personal, human, artisan | Callout notes, "from the author", decorative labels |

### Example: Mixed Heading

```css
h1 {
  font-family: var(--font-serif);
  font-weight: 700;
  letter-spacing: -0.02em;
}

.quote {
  font-family: var(--font-hand);
  font-size: 1.25rem;
  color: var(--ink-light);
}
```

### Loading Strategy

For privacy (no external tracking), consider self-hosting:
- Download Google Fonts as WOFF2 files
- Serve from local `/fonts/` directory
- Fall back to system fonts if unavailable

## 7. Iconography Evolution

### Rationale

Current icons are clean geometric glyphs. Solarpunk icons should feel illustrated — tiny drawings that could appear in a field guide, not a tech dashboard. Or alternatively, hand-drawn line icons with organic wobble.

### Icon Style Options

#### Option A: Illustrated Icons

Small illustrations within each icon:

- Sun icons with petal details
- Panel icons showing cells as seeds
- Compass as a flower compass
- Home as a house with growing garden
- Settings as gears with leaves

#### Option B: Organic Line Icons

Line icons with deliberate imperfection:

- Slightly wobbly lines (like hand-drawn)
- Organic end-caps (rounded, not square)
- Variable stroke width (thick where important)

#### Implementation

```css
/* Organic line icon style */
.icon {
  stroke-linecap: round;
  stroke-linejoin: round;
  fill: none;
  stroke-width: 1.8;
}

/* Subtle wobble animation */
@keyframes wobble {
  0%, 100% { transform: rotate(0deg); }
  25% { transform: rotate(1deg); }
  75% { transform: rotate(-1deg); }
}

.icon--living:hover {
  animation: wobble 2s ease-in-out infinite;
}
```

### Icon Set Recommendations

| Icon | Solarpunk Redesign |
|------|-------------------|
| Sun | Sun with petal rays, warm glow |
| Panel | Panel with visible cells/seeds |
| Compass | Flower compass with N petal |
| Location | House with garden/roots |
| Settings | Gear with vine growing through |
| Chart | Tree or root system |
| Download | Seed dropping into soil |
| Upload | Flower blooming upward |

## 8. Storytelling Layouts

### Rationale

Solarpunk is inherently narrative — it's about transformation, cycles, and connection to place. Layouts should tell visual stories, not just present information.

### Layout Patterns

#### Timeline: Sun's Journey

Show daily/seasonal data as a timeline:

|  Yield: 0kWh  |   Yield: 2kWh   |   Yield: 5kWh   |   Yield: 3kWh   |  Yield: 0kWh|

Show impact of orientation changes:

These improvements can be implemented incrementally. Suggested priority:

1. **Phase 1:** Expand color palette (low effort, high impact)
2. **Phase 2:** Paper textures (low effort, adds warmth)
3. **Phase 3:** Organic border radii (medium effort, distinctive)
4. **Phase 4:** Biophilic animations (medium effort, delightful)
5. **Phase 5:** Typography evolution (medium effort, personality)
6. **Phase 6:** Iconography evolution (higher effort, distinctive)
7. **Phase 7:** Organic data viz (higher effort, transformative)
8. **Phase 8:** Storytelling layouts (ongoing, content-dependent)
